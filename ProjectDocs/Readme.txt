2. 产品背景与业务目标
2.1 业务背景
企业存在大量碎片化轻量业务场景：资料采集、凭证上传、投保理赔、报销登记、线上申报、内部流程登记等，传统定制开发周期长、成本高、迭代慢。
企业内部系统烟囱化严重、数据孤岛、异构接口对接复杂，无统一集成标准。
图片上传、OCR 识别、重复去重、自动报表是高频刚需，传统工具配置复杂、无法资产沉淀复用。
通用大模型仅能对话，无法落地业务流程、生成可用页面、无合规审计，企业需要AI 规划 + 人工确认 + 可控可审计的半自动模式。
政企、金融、保险行业对审批流转、定时调度、多环境交付、API 开放、安全脱敏、权限隔离有强刚需。
行业 Agent 遵循 AgentSkills 规范，需兼容 OpenClaw 生态 Skill 双向导入导出，减少重复造轮子。
系统运行中必然产生配置异常、业务异常、代码 Bug，需要一套AI 自动捕获、主动告警、智能分析、自愈处置、代码级问题自动流转研发测试发版、经验沉淀自学习、知识生态积分运营的完整运维保障体系。
2.2 产品目标
打造「AI Agent + GenUI 动态界面 + 低代码编排 + Skill 原子复用 + 语义集成 + 多端分发 + 图像智能 + 智能运维 + AI 虚拟研发团队 + 知识生态运营」企业级轻量化末梢场景平台。
自然语言快速生成业务场景与页面，大幅降低开发交付周期；
沉淀行业模板、原子技能、故障知识库、测试用例资产库，开箱即用可复用；
完善审批、定时、事件触发、多环境迁移、API 开放，支撑 SaaS 与私有化交付；
满足金融、保险、政企数据隔离、合规审计、脱敏安全、等保要求；
遵循 AgentSkills 标准，与 OpenClaw 技能生态双向兼容；
搭建全链路异常捕获、多渠道告警、AI 智能分析、故障自愈、代码 Bug 自动闭环、知识沉淀共享 + 积分激励的完整体系。
3. 产品定位与业务边界
3.1 产品定位
AI Agent 驱动、GenUI 动态界面、Skill 原子化复用、多端全域分发、图像智能自动化、行业模板化、可 SaaS 可私有化、兼容 OpenClaw 生态、自带 AI 智能运维与 AI 虚拟研发测试发版团队、具备知识生态积分运营能力的企业轻量化末梢场景全生命周期治理平台。
重点服务：保险、金融科技、大中型政企、集团内部数字化。
3.2 包含业务范围
AI Agent：自然语言理解、多轮澄清、方案规划、技能调度、沙箱自测、人工确认上线；
GenUI 动态 UI：组件库管理、自然语言生成页面、增量编辑、页面数据持久化；
低代码可视化：拖拽编排、组件布局、样式定制、与 GenUI 双向兼容；
Skill 原子能力：基础交互、集成接口、图像智能、消息通知、审批、定时任务、API 开放、OpenClaw 兼容适配；
技能商店：公共 + 私有双仓、版本管理、权限隔离、审核上架、导入导出；
语义集成中心：多协议适配器、接口标准化、OCR / 报表模板、MCP 服务暴露；
场景治理：版本生命周期、灰度发布、依赖检测、分发配置、访问时效控制；
异常管理中心：全链路异常捕获、统一错误码、日志溯源、台账归档、强制结构化处置录入；
多端适配：PC/H5 / 企微 / 钉钉 / 小程序 / 原生 APP 统一适配；
图像智能链路：安全检测、格式校验、MD5/pHash 去重、OCR 识别、报表批量生成；
审批流程：单人 / 会签 / 或签、加签转派、超时多级流转、催办转交自动驳回；
定时 & 外部事件：Cron 定时、失败重试、并发控制、Webhook/MQ 事件过滤触发；
行业模板库：保险 / 政企标准模板、一键复制、AI 微调适配；
运营后台：租户管理、资源配额、用量统计、运营大盘、知识库积分管理；
多环境迁移：开发 / 测试 / 预发 / 生产隔离、配置迁移、敏感密钥自动脱敏；
API 开放平台：接口授权、APIKey+IP 白名单、限流鉴权、文档与调用日志；
OpenClaw 兼容层：Skill 导入导出、格式校验、字段映射、安全适配；
智能运维体系：全链路告警、AI 运维 Skill 归档分析、故障自愈、知识库沉淀；
AI 虚拟团队：需求分析 / 研发分析 / AI 代码编辑 / 自动化测试 / 发布运维六大内置 Skill；
AI 虚拟团队监控看板：任务队列、状态监控、成功率、Token 消耗、白名单配置、人工待办；
知识积分运营：租户故障案例匿名贡献、公共库审核、积分发放、权益兑换。
3.3 能力边界
不做重型复杂流程引擎，不支持多级深度嵌套分支；
不承载资金交易、清算账务、核心承保理赔等强金融核心业务；
AI Agent 仅辅助规划调度，核心配置与上线必须人工确认，禁止全自动越权；
OCR 依赖第三方服务商，平台仅做调度容错，不自研识别算法；
定时任务不支持自定义脚本，仅允许平台原子能力编排；
API 开放仅暴露标准化能力，不支持任意自定义接口开发；
不直接运行 OpenClaw 本地 Shell、浏览器自动化等高风险 Skill，仅做 HTTP 服务化适配；
AI 自动改码受控可控：仅允许白名单内模块修改，内核代码强制人工处理，重大变更必人工评审。
4. 整体架构与多租户设计
4.1 整体分层架构
AI Agent 智能中枢层：意图理解、多轮澄清、方案规划、技能编排、角色 Skill 调度；
GenUI 运行时层：组件仓库、UI 指令解析、页面渲染、租户隔离、操作审计；
AI 能力层：LLM 语义解析、故障分析、代码编辑、知识库推理、测试用例智能生成；
技能执行层：原子 Skill 调度、流程编排、分支循环、容错重试、事务补偿；
应用设计层：低代码画布、场景配置、模板管理、审批 / 定时配置、测试用例管理；
语义集成层：HTTP/SOAP/MySQL/MQ 适配器、接口标准化、MCP 服务注册；
网关接入层：路由转发、鉴权、限流、熔断、Webhook/MQ 请求适配；
场景运行层：页面服务、业务流程、图像任务、审批流转、定时调度；
多端适配层：PC、H5、小程序、企微钉钉、原生 APP 统一渲染；
管控治理层：版本管理、灰度发布、权限审计、异常捕获、测试用例版本管控；
运营监控层：租户用量、业务指标、接口性能、AI 虚拟团队监控、积分统计；
兼容适配层：OpenClaw Skill 解析、格式转换、安全扫描、导入导出；
智能运维层：全链路异常捕获、多渠道告警、故障知识库、自愈策略；
AI 虚拟团队层：六大内置角色 Skill，闭环代码 Bug 分析 / 改码 / 测试 / 发版；
知识运营层：私有 / 公共故障库、匿名脱敏审核、积分发放、权益兑换。
4.2 多租户隔离设计
租户强逻辑隔离：业务数据、场景配置、Skill 资产、模板、故障台账、私有知识库、自定义测试用例完全隔离；
部署形态：公有云 SaaS、专属租户、私有化单机 / 集群；
公共资源共享：公共组件、公共 Skill、行业模板、公共错误码、公共知识库、通用测试用例模板；
私有资源独享：租户自建场景、私有 Skill、私有故障库、自定义测试用例；
分发链接、二维码、API 密钥严格租户隔离，禁止跨租户越权访问；
多环境迁移配置隔离，敏感密钥自动脱敏，防止跨环境泄露；
OpenClaw 导入 Skill 归属当前租户，不可跨租户共享；
知识库贡献自动脱敏，仅共享匿名案例，不泄露租户私有敏感信息。
5. 全局通用规则
所有操作全程审计留痕：操作人、时间、IP、变更前后记录，日志保留≥180 天；
所有编辑自动云端草稿保存，支持断点续编、版本回溯；
高危操作强制二次确认：删除、下线、批量变更、Skill 导入导出、环境迁移、AI 白名单变更；
全站 HTTPS 传输，手机号 / 身份证 / 密钥等敏感字段传输存储自动脱敏；
图片上传自动内容安全检测，涉黄涉政可配置拦截或仅告警；
AI Agent 关键配置变更、版本上线必须人工确认，禁止全自动生效；
自定义 Skill 新增 / 版本变更必须机审 + 人工审核才可上架；
API 对外调用必须密钥 + 权限双重校验，异常调用自动限流告警；
定时任务、外部事件执行异常自动入异常中心、触发告警；
所有 Skill 遵循 AgentSkills 规范，导入导出强制格式校验与安全扫描；
自动化测试用例、AI 改码白名单均支持版本管理 + 导入导出 + 模板预置；
异常工单关闭必须完成结构化信息填写，否则无法闭环。
6. 核心模块详细需求
6.1 工作台
数据卡片：场景总数、上线数、访问量、接口成功率、异常数量、AI 生成量、OCR 调用、审批待办、AI 虚拟团队任务数、知识库积分；
待办中心：场景待发布、异常待处理、技能待审核、审批待办、AI 修复方案评审、公共知识库贡献审核；
快捷入口：新建 AI 场景、技能设计、语义集成、异常中心、技能商店、模板库、AI 虚拟团队监控看板、我的积分；
数据权限按租户 + 角色隔离，仅展示有权限内容。
6.2 AI 场景 & GenUI 动态界面
自然语言生成页面与业务流程，自动拆解组件、表单、流程节点；
支持存量页面自然语言增量修改，保留原有布局与数据；
生成历史、UI 指令日志全量留存，支持版本回放；
沙箱实时预览，人工确认后正式发布；
GenUI 组件行为可控，操作全程审计；
支持编排调用平台标准 Skill 及 OpenClaw 导入 Skill。
6.3 技能编排中心
可视化画布支持顺序、条件分支、循环、容错重试、子流程引用；
预置节点：图片上传、安全检测、去重、OCR、报表、接口调用、审批发起、消息通知、Webhook 触发；
支持页面事件、表单提交、定时、外部事件绑定技能流程；
OpenClaw 标准 HTTP 类 Skill 可直接拖拽编排；
流程执行日志全链路记录，异常自动标记入异常中心。
6.4 Catalog 组件管理
预置基础 / 表单 / 布局 / 业务 / 审批 / 图像组件；
支持自定义 Vue/React 组件注册上架、版本管理；
自定义组件需超管审核后全平台可用；
组件版本更新自动检测依赖场景，提示兼容校验。
6.5 低代码可视化编辑器
拖拽布局、样式配置、数据绑定、事件配置；
与 GenUI 页面双向兼容，可互相编辑调整；
支持 PC/H5 自适应布局、样式模板快速套用。
6.6 语义集成中心
多协议适配器：HTTP、HTTPS、SOAP、MySQL、PostgreSQL、RabbitMQ、Kafka；
接口标准化入参出参，适配全局统一错误码映射；
预置 OCR、短信、邮件、企微钉钉消息通用配置模板；
支持语义接口发布为 MCP 服务，供 AI Agent 自动调用；
适配 OpenClaw HTTP Skill 规范，自动字段映射。
6.7 场景治理中心
生命周期：草稿→编辑完成→沙箱测试→灰度发布→正式上线→停用下线；
每次编辑自动生成新版本，支持版本对比、一键回滚；
上线前自动依赖校验：组件、Skill、接口、模板有效性；
灰度支持按用户 ID / 租户范围放量；
生成访问链接、二维码、iframe 嵌入地址；
支持访问有效期、一次性访问、IP 绑定防盗用。
6.8 异常管理中心
全模块异常统一汇聚：前端渲染、业务流程、接口、图像、审批、定时、Skill 解析、代码 Bug 等；
按 6 位统一错误码分类，支持模块 / 错误码 / 时间 / 租户筛选；
异常详情：错误码、租户、链路 ID、请求参数、堆栈日志、发生时间；
支持状态标记、备注、关联故障案例；

6.10 图像智能全链路

8. 安全与合规规范
操作审计：所有配置、发布、迁移、Git 操作、白名单变更、知识库审核日志保留≥180 天；
数据脱敏：敏感个人信息、密钥、配置自动脱敏；租户知识库贡献自动去除租户标识；
访问安全：会话超时登出、密码复杂度、异地登录提醒、超管双因素认证；
接口安全：网关鉴权、防篡改、限流熔断、IP 白名单、请求签名；
UI 安全：XSS 过滤、禁止恶意脚本注入、GenUI 组件行为约束；
外部 Skill 安全：OpenClaw 导入高危能力扫描、禁止本地 Shell 与高权限操作；
跨环境迁移：生产密钥自动脱敏占位，目标环境手动补填；
Git 权限最小化：AI 仅可推送 ai-fix / 前缀分支，禁止操作 main/master；令牌有效期≤90 天、自动轮换；所有 Git 操作全审计；
AI 改码受控：仅白名单内模块可自动修改，白名单外自动转人工，不可绕过。

13. 风险识别与应对方案
大模型依赖：故障自动降级为模板模式，备用模型兜底；
OCR 服务商：多服务商自动切换，人工上传兜底；
数据泄露：租户隔离、脱敏、审计、知识库匿名贡献；
AI 乱改代码：白名单管控 + Git 分支限制 + 人工评审；
测试用例覆盖不足：预置通用模板 + AI 生成 + 用例版本管理；
知识库质量差：异常工单强制结构化录入；
知识无法共享：匿名脱敏贡献 +积分激励规则；
AI 成本不可控：Token 监控、预算告警、租户功能开关；
Git 越权操作：权限最小化、分支限制、全链路审计。
补充：知识库贡献积分运营规则
租户提交匿名故障案例，超管审核通过纳入公共库，单条奖励 50 积分；
积分永久累计不清零，仅限当前租户使用；
积分权益：兑换 AI 调用额度、高级模板、定时配额扩容、技术服务抵扣；
超管后台可配置单条积分值、兑换门槛、消耗标准；
积分变动全程日志，租户工作台可查明细与兑换记录。
