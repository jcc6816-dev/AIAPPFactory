# 通用增长模板目录 V1

更新时间：2026-05-21  
适用范围：互联网 / 终端用户 / 个人 / 小团队 / 小型组织  
不适用范围：企业行业深度流程模板

## 1. 模板分层说明

当前模板库分为两条线：

### A. 通用增长模板

面向更通用、更标准化的互联网和终端用户场景。

参考竞品：

- v0
- Typeform
- Jotform
- Fillout
- Tally
- Paperform

目标：

- 让用户不从 0 开始。
- 让首次体验更稳定。
- 让生成结果更像成熟产品。
- 让 Agent 有明确修改起点。

### B. 企业行业模板

面向保险、金融、政企、集团内部业务。

这部分必须基于线下调研，不在本文档中凭空设计。

## 2. 模板来源与边界

每个模板进入产品实现前，都需要标明来源，避免通用想象和企业真实流程混在一起。

来源类型：

- `public_benchmark`：基于公开竞品、公开模板库和通用互联网场景整理。
- `user_research`：基于真实用户访谈、客户反馈或线下调研整理。
- `internal_demo`：仅用于内部演示或销售演示，不代表行业最佳实践。

本文档当前只覆盖 `public_benchmark` 类型的通用增长模板。

企业行业模板必须等待业务调研输入后再单独沉淀，研发和 AI 编程助手不得凭空补全保险、金融、政企、集团内部审批等流程。

## 3. 首批模板选择原则

首批模板不追求数量，而追求高频、好理解、易演示。

选择标准：

- 用户一眼能理解用途。
- 可以直接发布使用。
- 表单字段不宜过多。
- 适合单题流填写体验。
- 能搭配 Agent 快捷任务修改。
- 后续可自然扩展 OCR / Webhook / 数据分析。

## 4. 模板分类与上线优先级

当前通用模板先分为 5 个大类：

| 分类 | 目标用户 | 首页展示价值 | 优先级 |
| --- | --- | --- | --- |
| 获客线索 | SaaS、服务商、个人品牌、咨询业务 | 最容易解释商业价值 | P0 |
| 活动报名 | 沙龙、直播、会议、社群活动 | 最容易演示发布和提交 | P0 |
| 反馈调研 | 产品、门店、课程、活动运营 | 最容易演示数据分析 | P0 |
| 预约咨询 | 课程、顾问、门店、本地服务 | 贴近个人和小团队 | P1 |
| 文件 / OCR 收集 | 报销、证件、资料、作品集 | 最能体现 OCR + Webhook 闭环 | P1 |

模板实现建议：

- `P0` 模板优先进入生成页和首页展示。
- `P1` 模板可先进入模板库，不一定全部放首页。
- 每个模板都必须有 Agent 快捷任务。
- 首页只展示 6 到 8 个代表性模板，避免用户选择疲劳。

## 5. 首批通用模板清单

用途：

用于官网、落地页、活动页收集潜在客户线索。

- 你的预算或计划时间？
- 还有什么想补充？

- `business`
- `minimal`

## 6. 首页展示推荐模板

首页不应展示全部模板，而应选择最能表达产品价值的 6 到 8 个模板：

- 线索收集表：证明获客价值。
- 活动报名表：证明发布和提交链路。
- 客户满意度调查：证明数据分析价值。
- 产品推荐问卷：证明互动体验价值。
- 预约咨询表：证明小团队和个人可用。
- 发票 / 票据收集：证明 OCR + Webhook 闭环。
- Beta 产品反馈：证明产品团队可用。
- 早鸟名单 / Waitlist：证明新产品启动场景。

首页模板卡片建议展示：

- 模板名称。
- 一句话用途。
- 字段数量。
- 推荐主题。
- 是否支持 OCR / Webhook。
- 一个“从此模板开始”按钮。

## 7. 暂不纳入首批的模板

以下模板有价值，但暂不作为首批通用模板：

- 支付订单表。
- 医疗问诊表。
- 金融贷款申请表。
- 保险理赔表。
- 政务材料收集表。
- 企业审批表。

原因：

- 涉及更复杂合规或行业流程。
- 字段和流程需要线下调研。
- 可能涉及敏感信息和权限设计。

## 8. 与 Agent 的关系

每个模板必须提供 Agent 快捷任务，避免用户从空白输入框开始。

模板进入生成页后，Agent 应先说明：

- 这个模板适合什么场景。
- 当前包含哪些关键字段。
- 可以如何快速修改。
- 是否建议接入 Webhook 或 OCR。

示例：

```text
我已基于“活动报名表”模板创建草稿。
你可以让我：
1. 改成高端沙龙报名
2. 增加饮食偏好
```
