# 通用增长模板首批清单 V1

更新时间：2026-05-20  
适用范围：互联网 / 终端用户 / 个人 / 小团队 / 小型组织  
不适用范围：企业行业深度流程模板

## 1. 模板分层说明

当前模板库分为两条线：

### A. 通用增长模板

面向更通用、更标准化的互联网和终端用户场景。

参考竞品：

- v0
- Typeform
- Jotform
- Fillout
- Tally
- Paperform

目标：

- 让用户不从 0 开始。
- 让首次体验更稳定。
- 让生成结果更像成熟产品。
- 让 Agent 有明确修改起点。

### B. 企业行业模板

面向保险、金融、政企、集团内部业务。

这部分必须基于线下调研，不在本文档中凭空设计。

## 2. 模板来源与边界

每个模板进入产品实现前，都需要标明来源，避免通用想象和企业真实流程混在一起。

来源类型：

- `public_benchmark`：基于公开竞品、公开模板库和通用互联网场景整理。
- `user_research`：基于真实用户访谈、客户反馈或线下调研整理。
- `internal_demo`：仅用于内部演示或销售演示，不代表行业最佳实践。

本文档当前只覆盖 `public_benchmark` 类型的通用增长模板。

企业行业模板必须等待业务调研输入后再单独沉淀，研发和 AI 编程助手不得凭空补全保险、金融、政企、集团内部审批等流程。

## 3. 首批模板选择原则

首批模板不追求数量，而追求高频、好理解、易演示。

选择标准：

- 用户一眼能理解用途。
- 可以直接发布使用。
- 表单字段不宜过多。
- 适合单题流填写体验。
- 能搭配 Agent 快捷任务修改。
- 后续可自然扩展 OCR / Webhook / 数据分析。

## 4. 首批通用模板清单

### 4.1 线索收集表

用途：

用于官网、落地页、活动页收集潜在客户线索。

推荐字段：

- 怎么称呼你？
- 你的联系方式是什么？
- 你对哪类产品或服务感兴趣？
- 你的预算或计划时间？
- 还有什么想补充？

推荐主题：

- `business`
- `minimal`

Agent 快捷任务：

- 改成适合 SaaS 官网获客。
- 增加预算范围选项。
- 检查字段是否太多。
- 生成销售跟进提示。

OCR / Webhook 建议：

- OCR：不需要。
- Webhook：适合推送到 CRM、飞书、钉钉。

### 4.2 联系我们表

用途：

用于网站、个人主页、机构页面接收咨询。

推荐字段：

- 怎么称呼你？
- 你的邮箱或电话？
- 你想咨询什么？
- 希望我们什么时候联系你？

推荐主题：

- `minimal`
- `business`

Agent 快捷任务：

- 改成适合个人咨询。
- 改成适合企业官网。
- 增加服务类型选项。
- 生成自动回复文案。

OCR / Webhook 建议：

- OCR：不需要。
- Webhook：适合推送到团队 IM 或客服系统。

### 4.3 活动报名表

用途：

用于会议、沙龙、直播、线下活动报名。

推荐字段：

- 怎么称呼你？
- 你的手机号或邮箱？
- 你所在公司或组织？
- 你的职位或身份？
- 是否有饮食或座位偏好？

推荐主题：

- `minimal`
- `business`
- `dark`

Agent 快捷任务：

- 改成高端沙龙报名。
- 加一个饮食偏好字段。
- 加一个同行人数。
- 生成报名成功提示文案。

OCR / Webhook 建议：

- OCR：通常不需要。
- Webhook：适合推送到活动运营群。

### 4.4 活动反馈表

用途：

用于活动结束后收集体验反馈和改进建议。

推荐字段：

- 你参加的是哪场活动？
- 你对本次活动整体满意吗？
- 哪个环节最有价值？
- 哪些地方需要改进？
- 是否愿意参加后续活动？

推荐主题：

- `minimal`
- `retro`

Agent 快捷任务：

- 改成适合线上直播反馈。
- 增加 NPS 评分。
- 精简成 3 个问题。
- 生成反馈总结维度。

OCR / Webhook 建议：

- OCR：不需要。
- Webhook：适合推送到运营复盘表或群消息。

### 4.5 客户满意度调查

用途：

用于产品、服务、门店或交付后的满意度调查。

推荐字段：

- 你使用的是哪项产品或服务？
- 你的整体满意度如何？
- 哪一项体验最好？
- 哪一项最需要改进？
- 是否愿意被进一步回访？

推荐主题：

- `business`
- `minimal`

Agent 快捷任务：

- 改成适合门店服务。
- 改成适合软件产品反馈。
- 增加 NPS 评分。
- 检查问题是否引导性太强。

OCR / Webhook 建议：

- OCR：不需要。
- Webhook：适合推送到客服或运营系统。

### 4.6 产品推荐问卷

用途：

用于根据用户偏好推荐产品、方案或服务。

推荐字段：

- 你主要想解决什么问题？
- 你的使用场景是什么？
- 你更看重哪些因素？
- 你的预算范围？
- 是否需要人工顾问联系？

推荐主题：

- `dark`
- `business`
- `brutalism`

Agent 快捷任务：

- 改成适合课程推荐。
- 改成适合软件方案推荐。
- 增加预算区间。
- 优化成更像互动测验。

OCR / Webhook 建议：

- OCR：不需要。
- Webhook：适合推送到销售或推荐系统。

### 4.7 预约咨询表

用途：

用于课程、服务、顾问、诊所、美容、线下门店预约。

推荐字段：

- 怎么称呼你？
- 你想预约哪项服务？
- 希望预约的日期和时间？
- 你的联系方式？
- 是否有特殊说明？

推荐主题：

- `minimal`
- `retro`
- `business`

Agent 快捷任务：

- 改成适合课程顾问预约。
- 改成适合门店服务预约。
- 增加可选时间段。
- 生成预约确认文案。

OCR / Webhook 建议：

- OCR：通常不需要。
- Webhook：适合推送到日程、客服或门店群。

### 4.8 招聘申请表

用途：

用于收集候选人信息和简历附件。

推荐字段：

- 你的姓名？
- 你的联系方式？
- 你想申请哪个岗位？
- 你的工作年限？
- 请上传简历。
- 有什么补充说明？

推荐主题：

- `business`
- `minimal`

Agent 快捷任务：

- 改成适合实习生招聘。
- 加一个作品集链接。
- 增加简历上传字段。
- 检查候选人信息是否足够筛选。

OCR / Webhook 建议：

- OCR：后续可用于简历解析。
- Webhook：适合推送到 HR 群、ATS 或表格。

## 5. 暂不纳入首批的模板

以下模板有价值，但暂不作为首批通用模板：

- 支付订单表。
- 医疗问诊表。
- 金融贷款申请表。
- 保险理赔表。
- 政务材料收集表。
- 企业审批表。

原因：

- 涉及更复杂合规或行业流程。
- 字段和流程需要线下调研。
- 可能涉及敏感信息和权限设计。

## 6. 与 Agent 的关系

每个模板必须提供 Agent 快捷任务，避免用户从空白输入框开始。

模板进入生成页后，Agent 应先说明：

- 这个模板适合什么场景。
- 当前包含哪些关键字段。
- 可以如何快速修改。
- 是否建议接入 Webhook 或 OCR。

示例：

```text
我已基于“活动报名表”模板创建草稿。
你可以让我：
1. 改成高端沙龙报名
2. 增加饮食偏好
3. 精简到 4 个字段
4. 生成报名成功文案
```

## 7. 后续实现建议

第一阶段实现建议：

- 先用代码内置模板数据，不急于建模板数据库。
- 模板数据结构要接近未来数据库结构。
- 生成页先支持模板选择和一键应用。
- Agent 先能读取模板的 `agentQuickActions`。

暂不实现：

- 模板市场。
- 用户自定义模板。
- 模板搜索排序。
- 模板付费分层。
- 模板版本管理。
