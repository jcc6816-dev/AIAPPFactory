# MVP 下一阶段路线图

更新时间：2026-05-09

## 目标

这份文档用于回答一个实际问题：

当前 MVP 已经可运行，但界面和体验仍有不少可打磨空间。此时如果继续深挖单页细节，容易陷入无限优化；如果直接往后加功能，又可能让当前页面返工。

因此需要先明确：

- 当前 MVP 已经完成到什么程度
- 哪些后续功能还会反向影响页面结构
- 接下来应该先做什么，后做什么
- 哪些页面现在只需要“可用”，哪些值得后续单独做专项

## 当前判断

当前更适合：

1. 先确定后续功能路线图
2. 再决定哪些页面值得继续精修

而不是：

1. 继续无限优化单个页面
2. 然后在后续功能叠加时被迫返工

## 当前 MVP 已完成能力

### 1. 基础链路

- Supabase 已接通
- `users / credits / forms / form_submissions / webhook_logs` 已真实写库
- `Development Sign In` 可用于本地演示

### 2. 表单生成与发布

- 一句话 AI 生成表单草稿
- 保存表单
- 表单列表页 / 新建页 / 详情页
- 分享链接
- 二维码分享

### 3. 分享填写体验

- 公开分享页已改为移动端优先单题流
- 中英文双语已接入
- 公开页已进行多轮 UI 提质

### 4. 提交与推送

- 表单提交可写入 `form_submissions`
- Webhook 配置和推送已打通
- 支持通用 Webhook / 飞书 / 钉钉 / 企业微信
- 支持关键词 / 签名等常见模式
- `webhook_logs` 已留痕

### 5. 商业限制

- 免费用户最多 1 个表单
- 限制已在后端生效

## 路线图更新说明

本轮范围调整后，第一阶段明确新增：

- OCR 识别
- DeepSeek 大模型接入

这意味着后续路线图不能再把 OCR 视为“远期扩展项”，而应把它当成当前 MVP 必做项之一。

## 为什么现在不适合继续深挖页面细节

有几个原因：

### 1. 后续功能还会改信息结构

以下能力一旦继续补，会直接影响当前控制台页面结构：

- 基础数据面板
- 更多发布配置
- 提交记录与日志联动
- 订阅与升级入口强化
- 主题系统完善

也就是说，当前页面还不是完全稳定的。

### 2. `forms/new` 还没有最终产品模式

虽然当前创建页已经往“创作工作台”靠拢，但它的最终形态仍然没有冻结：

- 是更像 `Typeform` 的成品优先模式
- 还是更像 `Tally` 的文档式创作模式
- 或者是更偏 `Fillout` 的“生成 -> 调整 -> 发布”模式

在这条产品决策彻底定下来之前，不值得对像素级细节投入太多。

### 3. 有些页面还只是“过渡态”

尤其是：

- `forms/new`
- `forms` 列表页
- 表单详情页

它们已经从“能用”进化到“有产品感”，但还没达到“可冻结的成熟设计稿状态”。

## 推荐路线

建议后续分为三段。

### 第一段：补齐剩余 MVP 功能

目标：
先把 MVP 从“可运行开发版”推进到“结构稳定的可交付版”。

优先级建议：

1. OCR 基础能力接入
   需要完成文件字段进入 OCR、OCR 结果写库、失败留痕与结果展示。

2. DeepSeek 模型接入
   需要完成模型提供方抽象、DeepSeek provider 接入与表单生成功能切换。

3. 基础数据面板
   不只展示表单列表，要让控制台首页或 Forms 首页有更清楚的数据反馈。

4. 主题系统继续收口
   当前已有 3 套主题，但更多体现在分享页。后面要明确后台如何展示主题差异、主题如何被选择和理解。

5. 部署与安全收口
   包括：
   - `.env` 清单
   - 数据库迁移说明
   - RLS 策略规划
   - Webhook 失败策略说明
   - OCR 与模型 provider 配置说明

6. 演示与验收资料
   让当前版本更容易给客户、同事或实施团队演示。

### 第二段：冻结信息架构

目标：
在继续大规模打磨 UI 前，先稳定“系统到底有哪些页面、页面关系是什么”。

需要冻结的内容包括：

- 控制台导航结构
- `forms` 列表页的信息层级
- `forms/new` 的创作模式
- 表单详情页的“发布中心”结构
- 日志 / 提交 / 配置之间的关系

只有这一步冻结了，后续 UI 精修才不会频繁返工。

### 第三段：再做系统化 UI 精修

目标：
这时再做一轮真正的品牌级润色，而不是边做功能边抠细节。

建议按优先级：

1. `forms/new` 专项重做
2. 分享页动画与微交互精修
3. 控制台整体视觉统一
4. 主题系统品牌化

## 页面分级建议

为了避免无限打磨，建议把页面分级处理。

### A 级：值得后续单独专项优化

- `forms/new`
- 公开分享页

原因：

- 这是客户最强感知的两个页面
- 一个影响创建体验
- 一个影响填写体验

### B 级：先保持结构正确，后续再统一精修

- `forms` 列表页
- 表单详情页
- Webhook 配置页
- 提交记录页

原因：

- 这些页现在更重要的是结构和信息清晰
- 视觉还能继续提，但不值得优先深抠

### C 级：暂时不投入大规模视觉精修

- 早期调试态页面
- 仅内部使用的辅助页
- 需要等功能继续明确的页面

## 当前最值得做的下一步

如果不继续陷入当前页面细节，最合适的下一步是：

1. 先补齐剩余 MVP 必做项
2. 同时把页面信息架构收口
3. 等结构基本稳定后，再针对 `forms/new` 做独立设计专项

## 不建议当前立刻做的事

当前不建议立刻进入：

- Skill / Workflow 平台化扩展
- 多租户
- 审批流
- 可视化拖拽编辑器
- 更远期的 AI 虚拟团队能力

原因很简单：

- 这些会显著扩散当前 MVP 范围
- 也会进一步打乱当前页面和导航结构

## 一句话建议

当前最合理的策略不是“继续无限优化当前 UI”，也不是“立刻跳到长期平台能力”，而是：

**先完成包含 OCR 与 DeepSeek 在内的剩余 MVP 路线图，冻结结构，再做一轮真正值回报的 UI 精修。**
