# MVP 完整功能清单

更新时间：2026-05-09

## 说明

这份清单用于统一当前第一阶段 MVP 的完整范围。

与前一版本相比，本轮范围明确新增两项高优先级能力：

- OCR 识别
- DeepSeek 大模型接入

因此，当前第一阶段 MVP 不再只是：

- AI 表单
- 分享填写
- Webhook
- Billing 限制

而是升级为：

**AI 表单 + OCR + DeepSeek + Webhook + Billing + 数据看板**

## 第一阶段完整范围

### A. 账户与基础环境

- 用户登录
  - 本地开发登录可用
  - 后续切正式 OAuth
- 用户数据写入
  - `users`
  - `credits`
- Supabase 数据库接入
- 基础环境变量与初始化说明

### B. AI 表单生成

- 一句话生成表单草稿
- 支持基础主题选择
  - `minimal`
  - `business`
  - `dark`
- 草稿保存
- 表单列表页
- 表单详情页
- 创建页工作台
- 中英文双语

### C. 表单发布与分享

- 分享链接
- 二维码分享
- 移动端优先单题流公开页
- 公开页主题表现
- 提交成功反馈

### D. 表单提交与存储

- `form_submissions` 写库
- 提交答案存储
- 文件字段支持
- 文件基础元数据记录
- 提交结果可查询

### E. OCR 识别

这是当前新增到第一阶段的核心能力。

第一阶段至少应完成：

- 支持图片 / PDF 类字段进入 OCR 链路
- 服务端 OCR 调用封装
- OCR 原始结果留痕
- OCR 结构化结果留痕
- OCR 失败状态记录
- OCR 结果能被后续提交记录 / Webhook 使用

建议第一阶段优先场景：

- 发票 / 票据识别
- 基础文本提取
- 简单字段结构化

第一阶段暂不要求：

- 复杂版面理解
- 高级多模板文档识别
- 图片去重
- 报表生成

### F. DeepSeek 大模型接入

这是当前新增到第一阶段的第二项核心能力。

第一阶段至少应完成：

- 模型提供方抽象层
- DeepSeek provider 接入
- AI 表单生成可切到 DeepSeek
- 调用配置支持环境变量注入
- 错误处理与降级策略

建议第一阶段接入方式：

- 不是只绑定 DeepSeek
- 而是抽象成 `LLM Provider`
- 第一阶段重点落地 `DeepSeek`

建议第一阶段 DeepSeek 使用场景：

1. 表单草稿生成
2. OCR 识别结果的结构化整理

### G. Webhook 推送

- 表单详情页配置 Webhook
- 平台优先配置方式
  - 通用 Webhook
  - 飞书机器人
  - 钉钉机器人
  - 企业微信机器人
- 关键词 / 签名等模式
- 真实推送
- 重试机制
- `webhook_logs` 留痕

### H. 基础数据面板

第一阶段至少应补齐：

- 表单总数
- 提交总数
- 最近提交
- Webhook 推送状态概览
- OCR 状态概览

这部分不要求复杂运营后台，但要让用户能看到系统正在发生什么。

### I. 商业限制与计费基础

- 免费用户最多 1 个表单
- 后端强校验
- 升级入口提示
- Credits 基础链路
- 提交扣费基础逻辑

第一阶段可保持简化，不要求完整复杂套餐系统。

### J. 文档、部署与验收

- 数据库初始化说明
- 增量迁移说明
- 环境变量说明
- MVP 验收清单
- 演示脚本
- 部署检查清单

## 当前仍不纳入第一阶段的内容

以下能力即使长期会做，当前仍不纳入第一阶段：

- Skill 平台化商店
- Workflow 可视化编排器
- 审批流
- 多租户
- 拖拽式低代码编辑器
- OpenClaw / MCP 服务
- AI 虚拟团队
- 异常中心 / AI 运维闭环
- 复杂图像智能链路

## 优先级建议

### P0

- AI 表单生成
- 分享填写
- 表单提交
- OCR 识别基础能力
- DeepSeek 接入
- Webhook 推送
- 商业限制

### P1

- 基础数据面板
- 主题系统收口
- 部署与安全收口
- 详情页 / 列表页信息架构稳定

### P2

- UI 精修专项
- 更完整的日志体验
- 更强的模型策略切换

## 一句话总结

当前第一阶段 MVP 的完整定义应更新为：

**AI 表单生成、移动端单题流填写、OCR 识别、DeepSeek 大模型接入、Webhook 推送、基础数据面板与订阅限制。**
