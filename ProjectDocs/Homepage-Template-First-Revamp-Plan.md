## 1. 改版目标

首页当前不应继续承担“完整介绍所有功能”的任务，而应变成用户的第一启动页。

目标：

- 一眼看懂产品做什么。
- 一眼知道从哪里开始。
- 一眼看到可直接使用的模板。
- 一眼理解表单提交后可以 OCR、结构化、Webhook 推送。
- 让用户在 30 秒内进入“选择模板 / 输入需求 / 创建表单”的动作。

一句话定位：

> AI AgentFactory 帮你从模板或一句话开始，生成可发布、可收集数据、可自动处理和推送的业务表单。

## 2. 借鉴对象与原则

参考对象：

- v0：首页强调“输入需求即可开始”，并展示可直接复用的模板与示例。
- Typeform：模板页强调选择模板后再修改，降低从 0 开始的压力。
- Jotform：模板数量和分类降低用户找场景的成本。
- Tally / Fillout / Paperform：强调快速构建、发布和集成。

借鉴原则：

- 借鉴信息结构，不照搬视觉。
- 首页不堆概念，优先展示用户动作。
- 模板卡片要具体，不写抽象口号。
- 首屏必须同时支持“输入 Prompt”和“从模板开始”。

## 3. 首页信息架构

### 3.1 首屏 Hero

首屏目标：

让用户立刻理解“我可以在这里生成一个业务表单，并马上发布使用”。

建议结构：

```text
AI AgentFactory

从一句话或模板开始，生成可运行的业务表单

创建线索收集、活动报名、反馈调研、票据收集等场景。
发布后可收集数据和文件，并自动完成 OCR、结构化处理与 Webhook 推送。

[输入框：描述你想创建的场景...]
[生成表单] [浏览模板]
```

首屏下方可展示 3 个可信能力标签：

- 单题流填写体验。
- OCR / DeepSeek 自动结构化。
- Webhook 推送到飞书、钉钉、企微或自有系统。

### 3.2 模板启动区

标题：

```text
不想从 0 开始？选一个模板，马上改成你的业务场景
```

首页展示 6 到 8 个模板：

- 线索收集表。
- 活动报名表。
- 客户满意度调查。
- 产品推荐问卷。
- 预约咨询表。
- 发票 / 票据收集。
- Beta 产品反馈。
- 早鸟名单 / Waitlist。

每个模板卡片包含：

- 模板名称。
- 一句话用途。
- 标签：`获客` / `活动` / `反馈` / `OCR` / `Webhook`。
- 字段数量。
- 推荐主题。
- CTA：`从此模板开始`。

不建议首页展示：

- 全部 20 个模板。
- 复杂筛选器。
- 企业行业深度模板。
- 模板市场入口。

### 3.3 产品闭环说明

目标：

让用户知道这不是普通表单生成器，而是数据进入系统后的自动处理闭环。

建议用 4 步卡片：

```text
1. 选择模板或描述需求
2. Agent 帮你修改字段和体验
3. 发布分享链接并收集数据 / 文件
4. OCR + DeepSeek 结构化，Webhook 自动推送
```

每一步只写一句人话，不写技术堆栈。

### 3.4 Agent 能力说明

目标：

避免用户误解 Agent 只是聊天框。

建议文案：

```text
每个页面都有自己的 Agent
生成页帮你改表单，数据页帮你看提交，发布页帮你查 Webhook。
```

展示 3 个例子：

- 生成页：`把邮箱字段改成选填`。
- 数据页：`总结最近提交情况`。
- 发布页：`解释最近 Webhook 失败原因`。

### 3.5 典型场景区

目标：

让不同用户快速找到自己。

建议展示 3 类：

- 市场与增长：线索收集、资料领取、Waitlist。
- 活动与社群：活动报名、活动反馈、社群申请。
- 数据与文件收集：发票票据、资料上传、招聘申请。

### 3.6 底部 CTA

建议文案：

```text
从一个模板开始，5 分钟搭好你的第一个 AI 数据入口

[浏览模板] [创建新表单]
```

## 4. 首页不做什么

当前阶段首页不做：

- 不做复杂行业解决方案墙。
- 不做企业级能力全量展示。
- 不做模板市场。
- 不做社区模板。
- 不做过重动画。
- 不做需要后端新增表的模板搜索。

原因：

- 当前目标是市场验证，不是完整平台官网。
- 首页越复杂，用户越难理解第一步。
- 模板体系还处于内置模板阶段，不能过早包装成市场生态。

## 5. 首页实现建议

当前实现状态：

- `Code/services/form-templates.ts` 已作为统一模板数据源，内置 20 个公共高频模板。
- 首页模板卡片与生成页模板入口读取同一份数据，避免首页和创建页不一致。
- 生成页已支持 `/forms/new?template=<template_id>` 自动应用模板。
- 模板草稿会带出推荐 `ocr_template` 和 `webhook_provider`，保存后发布页可继续按推荐平台配置。
- 生成页模板选择区已展示分类、适用场景、字段数量、OCR 与推送建议，帮助用户理解为什么选这个模板。

第一阶段实现：

- 使用内置模板数据，不新增数据库。
- 首页模板卡片从同一份模板常量读取，避免文案和生成页模板不一致。
- 首屏 Prompt CTA 指向 `/forms/new`，并可通过 query 参数携带 prompt。
- 模板 CTA 指向 `/forms/new?template=<template_id>`。
- 如果 query 参数暂不实现，先跳转到创建页，由创建页展示模板区。

建议新增或调整：

- `Code/services/form-templates.ts`：统一模板数据源。
- `Code/components/landing/template-gallery.tsx`：首页模板展示区。
- `Code/app/[locale]/(default)/page.tsx`：接入模板优先首页结构。
- `Code/components/forms/form-generator.tsx`：后续读取 `template` 参数并应用模板。

## 6. 首页验收标准

改版完成后，应满足：

- 新用户 5 秒内能说出产品做什么。
- 首屏有明确输入框或创建按钮。
- 首页能看到至少 6 个可理解模板。
- 每个模板都能解释适用场景。
- 点击模板能进入创建路径。
- 首页能表达数据处理闭环，而不只是“生成表单”。
- 不引入企业行业模板的未经调研假设。

## 7. 推荐实施顺序

1. 抽象统一模板数据源。
2. 让生成页可从模板创建草稿。
3. 首页展示精选模板并跳转创建页。
4. 调整首页首屏文案和 CTA。
5. 再做视觉精修。

这样做的原因：

- 先有模板数据，首页才有真实内容。
- 先跑通模板创建，首页 CTA 才不是装饰。
- 最后再做视觉，可以避免反复返工。
