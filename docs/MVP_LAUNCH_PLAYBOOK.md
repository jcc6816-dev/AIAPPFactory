# AI FormFactory MVP Launch Playbook

更新时间：2026-05-26

## 目标

这份文档用于当前 MVP 的上线前演练、客户演示和卖点包装。范围只覆盖 AI FormFactory 当前已经实现的 AI 表单生成、模板、发布、提交、Webhook、数据查看、支持反馈、管理员运营和支付配套能力。

不覆盖长期 PRD V2.8 中的工作流编排、Skill 商店、MCP 服务、多租户治理、AI 运维闭环等平台级能力。

## 一句话定位

AI FormFactory 是一个面向业务团队的 AI 表单生成与数据收集工具：从一句话或模板开始，快速生成可发布的业务表单，并把提交数据通过后台、Webhook 和运营台形成闭环。

## 核心演示故事

### 1. 从模板吸引用户

入口：

- 首页
- `/templates`
- `/templates/lead-capture`

演示重点：

- 模板详情页有独立 SEO 页面。
- 预览可以切换 Phone/Desktop、Step Flow/Long Flow、Theme、Visual FX。
- 默认展示单题流，突出“像 Typeform 一样填写”的体验。
- 点击 `Use This Template` 后进入创建流程。

验收标准：

- 模板页面能独立打开。
- 预览控件切换后视觉反馈正常。
- 使用模板后，模板配置能进入新建表单流程。

### 2. 从 AI 创建可发布表单

入口：

- `/forms/new`

演示重点：

- 用户可以用一句话生成表单。
- 创建页提供预览、主题、视觉效果、长表单/单题流等创作控制。
- 表单可以保存草稿，也可以进入发布链路。

验收标准：

- 登录用户能进入创建页。
- 生成或模板草稿有可编辑 schema。
- 保存后能在表单列表和详情中看到记录。

### 3. 发布并复制分享入口

入口：

- `/forms/[id]/publish`

演示重点：

- 发布页给出发布前检查。
- 显示发布状态、分享链接、二维码。
- 展示 Artifact 历史，让用户知道表单经历了哪些变化。
- Webhook 配置与日志入口在发布场景内形成闭环。

验收标准：

- 缺少标题、字段、选项或 Webhook URL 时能提示风险。
- 可发布表单能生成公开分享链接。
- 二维码和复制链接入口可见。

### 4. 公开填写与提交

入口：

- `/f/[shareCode]`

演示重点：

- 公开页面不要求登录。
- 手机端展示单题流体验。
- 提交后进入成功反馈。

验收标准：

- 已发布表单可以公开提交。
- 未发布或无效 share code 不能正常收集数据。
- 提交数据进入后台记录。

### 5. 数据查看与 Webhook 日志

入口：

- `/forms/[id]/submissions`
- `/forms/[id]/webhook-logs`

演示重点：

- 数据页可查看提交列表。
- AI Data Co-pilot 可以解释提交情况。
- Webhook 日志页显示推送状态、失败原因和重试入口。

验收标准：

- 新提交出现在数据页。
- Webhook 成功或 mock 记录可见。
- 失败日志可用于问题定位。

### 6. 用户反馈与管理员处理

入口：

- `/support`
- `/admin/support`
- `/admin`
- `/admin/health`

演示重点：

- 用户可提交反馈或问题。
- 管理员可查看并回复工单。
- 管理首页展示用户、表单、提交、收入、Webhook 失败、开放工单等运营指标。
- Health 页检查上线环境配置。

验收标准：

- 登录用户能提交反馈。
- 管理员能看到并回复工单。
- `/admin/health` 不暴露密钥明文，只显示配置状态。

## 对外卖点包装

### 可直接讲给客户的 5 个卖点

1. AI 生成：从一句话快速生成业务表单草稿。
2. 模板库：常见获客、活动、反馈、预约等场景可以从模板开始。
3. 高颜值预览：Phone/Desktop、Step Flow/Long Flow、Theme、Visual FX 让表单更容易打动用户。
4. 发布收集闭环：发布链接、二维码、公开填写、数据查看一体化。
5. 集成与运营：Webhook 日志、支持反馈、管理员看板、部署健康检查支撑真实上线。

### 谨慎表达的能力

- OCR 可以作为 MVP 集成点演示，但不要包装成完整文档智能处理平台。
- Webhook 可以包装成轻量集成能力，但不要承诺复杂工作流编排。
- Artifact 历史可以包装成“创作变化留痕”，暂不包装成完整版本管理系统。
- 当前后台是单租户运营视角，不包装成多租户企业管理后台。

## 发布判断

可以进入小范围试用的条件：

- 自动化测试和生产构建通过。
- `/admin/health` 没有 Required 级失败项。
- 注册/登录、创建、发布、公开提交、数据查看、Webhook 日志、用户反馈、管理员回复全部手动跑通。
- Stripe 测试支付至少完成一次 checkout 到订单落库或状态回显。
- `.env.example`、部署清单和隐私/服务条款已同步生产域名策略。

不建议正式公开投放的情况：

- 正式 OAuth 未配置。
- Stripe webhook 未联调。
- 生产数据库缺少 `support_tickets` 或 MVP 核心表。
- AI provider key 未配置且首页仍强宣传 AI 生成能力。
- 管理员邮箱未配置或后台权限未验证。

## 下一轮建议

下一轮不建议再大规模加视觉功能。优先做三件事：

1. 用真实生产配置跑一次完整人工验收。
2. 用 Stripe test mode 跑通支付后状态回显。
3. 梳理首页、Pricing、模板页文案，把卖点统一成“AI 生成 + 模板 + 发布收集 + Webhook + 运营闭环”。
